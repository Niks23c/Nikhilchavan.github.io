🛒 Amazon Sales Analysis (SQL Project)

🏠 Home 📂 Projects

🧩 Problem Statement

Amazon wants to better understand its transactional retail data to improve sales, customer retention, and operational efficiency. The goal of this project is to use SQL-only techniques to:

  • Create new features from the raw data
  • Explore data patterns across cities, product lines, and customer segments
  • Generate actionable business insights

📁 Dataset Description

  • Source: Simulated Amazon retail sales data
  • Size: 1,000+ rows, 17 columns
  • Key Fields: Invoice ID, Branch, City, Customer Type, Gender, Product Line, Unit Price, Quantity, Total, COGS, Tax5, Date, Time, Payment Method, Rating

🛠️ Tools & Technologies

Tools: MySQL / SQL

SQL Techniques: GROUP BY, ORDER BY, CASE, RANK(), ROW_NUMBER(), CTEs, JOINs, Window Functions, ALTER, UPDATE, IS NULL

🔧 Feature Engineering

  • Extracted day and month from date
  • Created TimeOfDay column
  • Weekend flag added
  • Calculated gross margin and rating-based sales tag

📊 Exploratory Data Analysis (EDA)

Insights include:

  • Top revenue-generating product line and cities
  • Customer type revenue impact
  • Branch-level above-average sales
  • Gender-product preference
  • Branch-wise peak rating days

🏆 Key Insights & Recommendations

  • Food and Beverages is top-selling product line
  • Naypyitaw branch leads in revenue — potential for expansion
  • Member customers generate more sales — loyalty programs work
  • Gender-specific product strategies can improve sales
  • Optimize staffing based on branch-specific high-rating days
